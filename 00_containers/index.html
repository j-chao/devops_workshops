“DevOps is a tool-centric philosophy that supports a Continuous Delivery value chain.”
Delivery = deployment to Production and maintaining Production.
### Traditional Methods of deploying applications Typically virtualize the hardware layer (i.e. virtual server) Entangled executable, libraries, configuration and lifecycle with host OS VMs tend to be heavier, approx 600MB to several GB Hypervisor layer can take up to 33% of CPU overhead ### Container based approaches Operating system virtualization (i.e. virtual Linux machine) Encapsulated executables, libraries, configuration, and lifecycle Containers tend to be lighter, approx. 20MB to several hundred MB. Container layer can typically take between 2-5% CPU overhead popularity of Java JVM
# Container Advantages - Portable - Isolated - Lighter footprint & overhead (vs VMs) - Simplify DevOps practices - Speed up Continuous Integration - Empower Microservice architectures and adoption
